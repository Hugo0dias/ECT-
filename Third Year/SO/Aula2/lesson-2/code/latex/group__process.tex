\doxysubsection{Detailed Description}
\mbox{\hyperlink{group__process_process}{Process}}, \mbox{\hyperlink{group__process_sysV}{System-\/V}}, and \mbox{\hyperlink{group__process_posix_sem}{POSIX libraries}} wrapper module. 

This module removes defensive programming approach of native libraries.

All implemented functions, have exactly the same arguments and/or result of the original function, with the exception of returning an error indication.

Errors are handled by the implementation of two policies\+:
\begin{DoxyEnumerate}
\item EXIT\+\_\+\+POLICY (default)\+: describes the failed call in {\ttfamily stderr} (with the identification of the errno error, and the precise location the call), generates a segmentation fault (enabling a stack trace within a debugger like {\ttfamily gdb}), and exits program execution;
\item EXCEPTION\+\_\+\+POLICY\+: throws a {\ttfamily int} exception with the (errno) status error returned by the original function.
\end{DoxyEnumerate}

{\bfseries{System V group of IPC mechanisms}}

\label{group__process_sysV}%
\Hypertarget{group__process_sysV}%
 There are three resources for IPC\+: \mbox{\hyperlink{group__process_shm}{shared memory}}, \mbox{\hyperlink{group__process_sem}{semaphores}}, and \mbox{\hyperlink{group__process_msg}{message queues}}.

Common to all, is resource handling through a non-\/negative integer identifier. After proper creation and initialization, different processes can establish communication using the same integer identifier.

A key ({\ttfamily key\+\_\+t}) is used to establish this common identifier. There are three possibilities to define a key\+:


\begin{DoxyEnumerate}
\item {\ttfamily IPC\+\_\+\+PRIVATE}\+: In this case an alternative channel to communicate the identifier between processes is necessary (parent/child fork, file system, ...).
\item A fixed predetermined key number (may collide with other existing keys).
\item {\ttfamily ftok} function to generate a key from a path and a byte integer.
\end{DoxyEnumerate}

Usage\+:


\begin{DoxyItemize}
\item create new identifier\+: {\ttfamily msgget}/{\ttfamily semget}/{\ttfamily shmget} with a key and {\ttfamily IPC\+\_\+\+CREAT} and {\ttfamily IPC\+\_\+\+EXCL} flags
\item get existing identifier\+: {\ttfamily msgget}/{\ttfamily semget}/{\ttfamily shmget} with a key and other arguments as zero
\end{DoxyItemize}
