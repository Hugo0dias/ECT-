\doxysubsection{Detailed Description}
Design-\/by-\/\+Contract module. 

This module gives a better support for DbC in C++.

When compared with {\ttfamily assert.\+h} this module offers the following functionalities\+:
\begin{DoxyItemize}
\item separates different assertion types (preconditions, postconditions, invariants, and others);
\item augments assert with error message strings;
\item implements two error handling policies\+:
\begin{DoxyEnumerate}
\item EXIT\+\_\+\+POLICY (default)\+: describes the failed assertion in {\ttfamily stderr} (with the identification and the precise location of the assertion), generates a segmentation fault (enabling a stack trace within a debugger like {\ttfamily gdb}), and exits program execution;
\item EXCEPTION\+\_\+\+POLICY\+: throws a {\ttfamily char$\ast$} exception with the description of the failed assertion.
\end{DoxyEnumerate}
\end{DoxyItemize}

Debug settings\+:
\begin{DoxyItemize}
\item As happens with {\ttfamily assert.\+h}, this module accepts {\bfseries{{\ttfamily NDEBUG}}} macro definition to disable all assertion run-\/time verification.
\item Also, each assertion type verification can be disabled with macros\+: {\bfseries{{\ttfamily NDEBUG\+\_\+\+CHECKS}}}, {\bfseries{{\ttfamily NDEBUG\+\_\+\+PRECONDITIONS}}}, {\bfseries{{\ttfamily NDEBUG\+\_\+\+POSTCONDITIONS}}}, and {\bfseries{{\ttfamily NDEBUG\+\_\+\+INVARIANTS}}}.
\item A special (high priority) macro named {\bfseries{{\ttfamily DEBUG\+\_\+\+ONLY\+\_\+\+PRECONDITIONS}}} ensures that only precondition debugging remains active (regardless of other debug definitions).
\end{DoxyItemize}
